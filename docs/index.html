WebCore 2.0 Documentation

In Marrow Open Source Collective projects we aim to have our code be sufficient documentation. This collection is temporary until something more permanent can be arranged.

web.core

  • web.core.application — the primary WebCore WSGI application
  • web.core.compat — compatibility helpers bridging Python 2 and Python 3 differences
  • web.core.context — context attribute access dictionary
  • web.core.dispatch — dispatch registry and protocol adaption
  • web.core.extension — extension registry and management
  • web.core.util — additional utilities for working with core
  • web.core.view — view registry

web.app

  • web.app.static — delivery of static files from on-disk, primarily for development use

web.ext

  • web.ext.analytics — record performance statistics
  • web.ext.annotation — utilize Python 3 function annotations for typecasting
  • web.ext.base — construct request and response objects, and register default views
  • web.ext.debug — an interactive web-based REPL shell and capture of unhandled excpetions for interactive debugging
  • web.ext.local — thread local access to the context for those situations where such a solution just can't be avoided
